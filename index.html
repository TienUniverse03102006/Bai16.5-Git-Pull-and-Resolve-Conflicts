Một dự án có nhiều người làm và họ có thể sửa

    Trên máy ông A

  1. Đăng bài lên github và deploy như thường
  2. Trên github vào settings ->collaborators->enter password
  3. Add people

    Trên máy ông B

  1. Vào github, bấm vào ảnh đại diện góc phải màn hình
  2. Đến organizations-> accept
  3. Tạo một thư mục, chưa tạo file vội
  4. Trên Vscode, open folder vừa tạo trên Vscode
  5. Đến terminal, xong gõ git init
  6. git remote add origin 'địa chỉ file của ông A'
  7. git branch -m main
  8. git pull origin main

    Khi A và B đều sửa cùng 1 chỗ, thì ở B phải:

  1. git add .
  2. git commit -m "b sửa lần 1"
  3. git pull origin main
  4. Sau đó sẽ xuất hiện conflict giữa cái A và B đang sửa ở chỗ đó
  5. Chọn giữ lại cái của A hoặc B hoặc cả hai:
    1. accept current change: chấp nhận thay đổi mà máy B sửa
    2. accept incoming change: chấp nhận cái A sửa
    3. accept both changes: chấp nhận cả 2
  6. sau đó nhấn resolve in merger editor->complete change
  7. xong nhấn commit để hoàn tất việc merge

A có thể nhận được những thay đổi từ B khi B thêm bằng cách

  1. Nhấn vào biểu tượng số 3
  2. Chọn sync changes
